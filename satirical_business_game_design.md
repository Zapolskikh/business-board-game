# Сатирическая бизнес-игра: дизайн-документ для разработки

Дата версии: 2026-07-07  
Статус: рабочий дизайн-документ для MVP / прототипа  
Цель документа: передать агенту разработки в VS Code текущий концепт игры, сохраненные правила, роли, объекты и рекомендации по первой технической реализации.

---

## 1. Короткое описание игры

Это сатирическая настольная / цифровая бизнес-игра про деньги, власть, криминал, репутацию, риск и социальный лифт.

Игроки перемещаются по полю из трех концентрических кругов, покупают объекты, получают деньги и опыт, берут временные роли и используют их для изменения эффектов клеток. Главная фишка игры — не просто покупка объектов, а конкуренция за временные уникальные роли и их использование в подходящий момент.

Игра должна быть достаточно простой для понимания, но с живыми PvP-взаимодействиями: снятие ролей, скандалы, крыша, отжим объектов, аукционы, откаты в больницу/тюрьму, рискованные схемы.

## 2. Рекомендуемая первая техническая стратегия

### 2.1. Что проще для первых этапов

Для раннего прототипа, тестов правил и MVP с друзьями проще начать с веб-версии:

**Браузерный клиент + backend + серверная логика игры.**

Рекомендованный первый стек:

- Frontend: TypeScript + React или простой HTML/CSS/TypeScript.
- Rendering: сначала обычный DOM/SVG/Canvas, без сложного движка.
- Backend: Python FastAPI или Node.js.
- Real-time: WebSocket.
- Game state: JSON-состояние партии.
- Storage: сначала in-memory + сохранение JSON-файла, позже PostgreSQL.
- Deployment: один небольшой VPS или Render/Fly.io/Railway/Vercel + backend отдельно.

### 2.2. Почему не Godot сразу

Godot хорош для финального визуального клиента, Steam-версии и более “игрового” UI, но для текущего этапа он может замедлить разработку правил.

На первых этапах важнее быстро тестировать:

- экономику;
- баланс кругов;
- роли;
- клетки;
- порядок хода;
- PvP-действия;
- edge cases;
- ботов/автотесты;
- логи партии;
- админ-панель для изменения цифр.

Веб-прототип быстрее даст возможность:

- отправить ссылку друзьям;
- играть без установки;
- быстрее править правила;
- логировать каждое действие;
- запускать симуляции;
- делать внутренние инструменты баланса.

### 2.3. Когда переходить к Godot

Godot имеет смысл подключать позже, когда:

- правила стабилизированы;
- понятен UX;
- понятна экономика;
- есть рабочий backend;
- есть JSON API / WebSocket protocol;
- есть тестовые партии, подтверждающие, что игра интересная.

Тогда Godot можно сделать как отдельный красивый клиент, который подключается к тому же backend.

### 2.4. Важный принцип архитектуры

Логика игры должна жить не в UI, а в отдельном game engine module.

Правильная структура:

```text
game-engine/
  rules/
  actions/
  cells/
  roles/
  state/
  validators/
  simulations/

backend/
  websocket/
  lobby/
  rooms/
  persistence/

frontend/
  board/
  player-panel/
  action-panel/
  log/
```

Frontend или Godot-клиент должны только отправлять действие:

```json
{
  "game_id": "abc",
  "player_id": "p1",
  "action": "ROLL_DICE"
}
```

А backend/game engine должен возвращать новое состояние:

```json
{
  "state": "...",
  "events": ["Player rolled 5", "Player landed on Bank", "Player received bonus"]
}
```

## 3. Основные параметры игры

### 3.1. Игроки

- Максимум игроков: 6.
- Рекомендуемое количество игроков: 4.
- Боты желательны позже для тестов и одиночной игры.

### 3.2. Поле

Поле состоит из 3 концентрических кругов:

- 1 круг: базовый, безопаснее, стартовая экономика.
- 2 круг: средний риск и награда.
- 3 круг: высокий риск, высокая награда, меньше транспорта.

Сложность кругов регулируется не отдельной логикой каждого кольца, а коэффициентами:

- деньги за Start;
- цена объектов;
- размер штрафов;
- бонусы;
- стоимость услуг;
- риск/награда.

Поле может быть псевдорандомным, но некоторые клетки должны быть закреплены на определенных кругах/слотах.

### 3.3. Старт / прохождение круга

Каждый проход через Start дает игроку деньги. Размер зависит от уровня круга.

Примерная идея:

```text
Start bonus ring 1 < Start bonus ring 2 < Start bonus ring 3
```

Точные цифры балансируются тестами.

### 3.4. Деньги и опыт

В игре есть минимум два основных ресурса:

- деньги;
- опыт.

Деньги нужны для покупки объектов, услуг, штрафов, ставок и защиты.  
Опыт нужен для прогрессии и, вероятно, переходов между кругами.

### 3.5. Банкротство

Банкротство не должно выкидывать игрока из игры. Оно должно быть откатом/наказанием.

Идея:

```text
Банкротство = потеря части прогресса / объектов / откат, но не вылет из партии.
```

Детальные правила банкротства еще нужно определить.

## 4. Роли

### 4.1. Главное правило ролей

Роли временные, уникальные и эксклюзивные.

Если роль занята одним игроком, другой игрок не может просто взять эту роль, пока она не освободится или не будет снята/заблокирована отдельной механикой.

Главная механика игры — борьба за роли, их своевременное получение, защита, снятие и использование под текущие активы, маршрут и ситуацию на поле.

### 4.2. Список ролей

#### Капиталист

Темы:

- деньги;
- активы;
- инвестиции;
- покупка преимуществ;
- легальное решение проблем через деньги.

Уязвимости:

- налоги;
- проверки;
- засады;
- репутационные риски;
- силовые клетки.

#### Мафиози

Темы:

- крыша;
- давление;
- отжим;
- криминальные схемы;
- токсичные PvP-механики.

Уязвимости:

- госучреждения;
- блокпост;
- тюрьма;
- Военный;
- проверки.

#### Политик

Темы:

- субсидии;
- госучреждения;
- связи;
- снятие скандалов;
- роль, сильно завязанная на репутацию.

Уязвимости:

- скандалы;
- Газета;
- Журналист;
- потеря роли через 2 скандала.

#### Журналист

Темы:

- Газета;
- скандалы;
- расследования;
- публичное давление;
- снятие/блокировка ролей через репутацию.

Уязвимости:

- деньги и силовые угрозы скорее через общие механики.

#### Аферист

Темы:

- рискованные схемы;
- фальшивые документы;
- обход оплаты;
- манипуляции;
- высокий риск / высокая награда.

Уязвимости:

- тюрьма;
- блокпост;
- госучреждения;
- провалы на кубике.

#### Военный

Темы:

- силовой контроль;
- проверки;
- снятие ролей;
- борьба с Мафиози и Аферистом;
- блокпост.

Уязвимости:

- слабее по базовой экономике;
- сила проявляется через редкие силовые клетки.

## 5. Ключевые дизайн-правила

### 5.1. Ролевые эффекты только при активации клетки

Роль должна менять эффект клетки только в момент, когда игрок наступил на эту клетку.

```text
Наступил на клетку -> смотрим текущую роль игрока -> применяем соответствующий эффект.
```

### 5.2. Эффекты владения не зависят от роли

Если объект покупаемый, его эффект владения одинаковый для всех владельцев и не зависит от текущей роли владельца.

Это сделано, чтобы не плодить хаос:

- не нужно пересчитывать пассивные бонусы;
- не нужно помнить, кто кем был в момент покупки;
- игроку проще понять, почему что-то произошло;
- backend проще тестировать.

### 5.3. Структура объекта

Каждый объект желательно описывать так:

```text
Название

Тип:
- покупаемый объект / интерактивная клетка / опасность / сервис / место отката

Можно купить:
- да / нет

Теги ролей:
- роли, для которых есть особый эффект при наступлении

Эффект владения:
- только для покупаемых объектов
- не зависит от роли владельца

Базовая активация:
- что происходит с обычным игроком

Ролевая активация:
- что меняется, если наступивший игрок имеет профильную роль
```

## 6. Глобальные статусы и механики

### 6.1. Крыша

Крыша — расходуемая защита.

Используется в нескольких механиках:

- Засада: если есть Крыша, снимается Крыша; если нет — Больница.
- Блокпост: если эффект должен снять роль, но у цели есть Крыша, снимается Крыша вместо роли.
- Охранное агентство: позволяет получить/купить Крышу.

Крыша должна быть ценным расходником, особенно на верхних кругах.

### 6.2. Скандалы

Скандалы — репутационный счетчик.

Общее правило:

```text
2 скандала = потеря активной роли.
После потери роли скандалы обнуляются.
```

Если у игрока нет активной роли и он получает 2 скандала:

```text
Игрок получает стандартный штраф, скандалы обнуляются.
```

Газета — основной объект, который работает со скандалами.

### 6.3. Больница

Больница — отдельная клетка/место на первом круге.

Если игрок отправляется в Больницу:

- он перемещается на клетку Больница на первом круге;
- ход не пропускается;
- это пространственный откат, а не ожидание.

Больница отличается от Тюрьмы тем, что не обязана снимать роль.

### 6.4. Тюрьма

Тюрьма — отдельная клетка/место на первом круге.

Если игрок отправляется в Тюрьму:

- он перемещается на клетку Тюрьма на первом круге;
- теряет опыт;
- теряет активную роль;
- ход не пропускается.

Тюрьма — более жесткое юридическое наказание, чем Больница.

## 7. Сохраненные объекты и клетки

## 7.1. Охранное агентство

Тип: интерактивная клетка / сервис.  
Покупается: нет.  
Теги ролей: Мафиози, Военный, Капиталист.

Смысл:

- Крыша;
- снятие крыши;
- страховка объекта.

Базовая активация:

```text
Игрок может купить Крышу за деньги.
Крыша защищает от следующей Засады или от снятия роли через некоторые силовые эффекты.
```

Мафиози:

```text
Получает Крышу бесплатно.
Если Крыша уже есть, ничего дополнительно не происходит.
```

Военный:

```text
Может снять Крышу с выбранного игрока.
```

Капиталист:

```text
Может застраховать один свой объект.
Застрахованный объект остается за игроком до конца игры и не может быть отобран никаким способом.
```

Дополнительно:

```text
Охранное агентство нельзя купить во владение.
Покупаемые объекты должны быть более логичными для владения и очевидного бонуса.
```

---

## 7.2. Газета

Тип: покупаемый объект.  
Теги ролей: Журналист, Капиталист.  
Смысл: скандалы и потеря роли через репутацию.

Общее правило скандалов:

```text
2 скандала = потеря активной роли.
После потери роли скандалы обнуляются.
Если роли нет, при 2 скандалах игрок получает стандартный штраф, скандалы обнуляются.
```

Эффект владения:

```text
Если владелец наступает на свою Газету:
- снять 1 скандал.

Если другой игрок наступает на Газету:
- этот игрок получает 1 скандал.
```

Базовая активация:

```text
Если Газета свободна:
- можно купить Газету.

Если Газета принадлежит другому игроку:
- получить 1 скандал.
```

Журналист:

```text
Не получает скандал от чужой Газеты.
Вместо этого выбирает другого игрока, который получает 1 скандал.
```

Капиталист:

```text
Если наступил на чужую Газету, может заплатить владельцу и не получать скандал.
```

Политик:

```text
Без отдельного спецэффекта.
Газета уже полезна Политику через снятие скандалов и защиту роли.
```

---

## 7.3. Казино

Тип: покупаемый объект.  
Теги ролей: Аферист, Мафиози, Капиталист.  
Смысл: азартная денежная клетка; владелец в выигрыше по вероятности.

Базовая активация:

```text
Если Казино свободно:
- можно купить Казино.
- если не покупаешь, играешь против банка.

Если Казино принадлежит другому игроку:
- брось кубик.
- 1-4: проигрываешь деньги владельцу.
- 5-6: владелец выплачивает тебе выигрыш.
```

Если владелец наступает на свое Казино:

```text
Ничего не происходит по умолчанию.
```

Аферист:

```text
3-6 = выигрыш.
1-2 = проигрыш.
```

Мафиози:

```text
При проигрыше платит только 50% суммы.
```

Капиталист:

```text
Перед броском может удвоить ставку.
При победе получает двойной выигрыш.
При поражении платит двойной проигрыш.
```

---

## 7.4. Вокзал / Сеть вокзалов

Тип: покупаемый объект.  
Теги ролей: Аферист, Капиталист, Мафиози.  
Смысл: платное перемещение между вокзалами внутри одного круга.

Расположение:

```text
1 круг: 3 Вокзала
2 круг: 2 Вокзала
3 круг: 0 Вокзалов
```

Базовая активация:

```text
Если Вокзал свободен:
- можно купить Вокзал.

Игрок может оплатить проезд и переместиться на другой Вокзал этого же круга.

Если Вокзал принадлежит другому игроку:
- стоимость проезда получает владелец.
```

Эффект владения:

```text
Владелец получает стоимость проезда, когда другой игрок пользуется его Вокзалом.
Владелец может бесплатно перемещаться через свой Вокзал.
```

Аферист:

```text
Может пользоваться Вокзалом бесплатно, даже если он принадлежит другому игроку.
```

Капиталист:

```text
Может вместо поездки принудительно выкупить чужой Вокзал за 2x базовой цены.
Деньги получает текущий владелец.
```

Мафиози:

```text
Может вместо поездки бросить кубик.
6: отбирает чужой Вокзал бесплатно.
1-5: ничего не происходит.
```

## 7.5. Банк

Тип: интерактивная клетка / сервис.  
Покупается: нет.  
Теги ролей: Капиталист, Политик, Аферист.  
Смысл: моментальный денежный бонус, похожий на Start.

Правило:

```text
В Банке нет долгосрочных кредитов, долгов и счетчиков.
Банк должен быть моментальной бонусной клеткой.
```

Базовая активация:

```text
Игрок получает банковский бонус сразу.
Размер бонуса зависит от текущего круга и балансируется коэффициентом.
```

Капиталист:

```text
Получает банковский бонус x2.
```

Политик:

```text
Получает банковский бонус и снимает 1 скандал.
```

Аферист:

```text
Бросает кубик:
1-3: отправляется в Тюрьму.
4-6: получает банковский бонус x2.
```

Мафиози / Журналист / Военный:

```text
Базовый эффект.
```

---

## 7.6. Блокпост

Тип: интерактивная опасная клетка.  
Покупается: нет.  
Теги ролей: Военный, Мафиози, Аферист.  
Смысл: редкая, но жесткая силовая проверка.

Общее правило:

```text
Если Блокпост должен снять роль, но у игрока есть Крыша:
- снимается Крыша вместо роли.
```

Базовая активация:

```text
Игрок проходит проверку и платит небольшой штраф.
```

Военный:

```text
Выбирает любого игрока.
Этот игрок теряет активную роль.
Если у цели есть Крыша, снимается Крыша вместо роли.
```

Мафиози:

```text
Заплати крупный штраф или потеряй роль Мафиози.
Если есть Крыша, можно потерять Крышу вместо роли.
```

Аферист:

```text
Заплати крупный штраф или потеряй роль Аферист.
Если есть Крыша, можно потерять Крышу вместо роли.
```

Капиталист / Политик / Журналист:

```text
Базовая проверка и небольшой штраф.
```

---

## 7.7. Засада

Тип: интерактивная опасная клетка.  
Покупается: нет.  
Теги ролей: Мафиози, Военный, Капиталист.  
Смысл: проверка наличия Крыши; без Крыши верхние круги опасны.

Базовая активация:

```text
Если у игрока есть Крыша:
- снять Крышу.

Если Крыши нет:
- отправить игрока в Больницу.
```

Мафиози:

```text
Если есть Крыша:
- снять Крышу.

Если Крыши нет, бросить кубик:
1-3: Больница.
4-6: отбился, ничего не происходит.
```

Военный:

```text
Если есть Крыша, может выбрать: потерять Крышу или бросить кубик.

Бросок кубика:
1-2: Больница.
3-6: отбился, ничего не происходит.

Если Крыши нет, использует этот же бросок.
```

Капиталист:

```text
Если есть Крыша:
- снять Крышу.

Если Крыши нет:
- может заплатить выкуп и не идти в Больницу.
- если не платит или не может платить, отправляется в Больницу.
```

Журналист / Политик / Аферист:

```text
Базовый эффект.
```

## 7.8. Больница

Тип: отдельная клетка / место отката.  
Покупается: нет.  
Теги ролей: нет.  
Расположение: только на первом круге.

Эффект:

```text
Если игрок отправляется в Больницу:
- он перемещается на клетку Больница на первом круге.
- ход не пропускается.
```

Стратегический смысл:

```text
Без Крыши ходить на верхних кругах опасно, потому что Засада может откатить игрока в Больницу на первый круг.
```

## 7.9. Госучреждение

Тип: интерактивная клетка / сервис / проверка.  
Покупается: нет.  
Теги ролей: Политик, Капиталист, Аферист, Мафиози.  
Смысл: официальная бюрократия, субсидии, защита, проверки.

Базовая активация:

```text
Заплати административный сбор.
```

Политик:

```text
Выбери одно:
1. Получить субсидию.
2. Снять с себя 1 скандал.
```

Капиталист:

```text
Заплати фиксированную сумму и выбери одно:
1. Получить Крышу / легальную защиту.
2. Снять с себя 1 скандал.
```

Аферист:

```text
Может ничего не делать и пройти базовую активацию.
Или попытаться притвориться Политиком:

Брось кубик:
1-2: провал, Тюрьма.
3-6: успех, получить субсидию.
```

Мафиози:

```text
Заплати крупный штраф или отправляйся в Тюрьму.
```

Журналист / Военный:

```text
Базовая активация.
```

## 7.10. Такси

Тип: интерактивная клетка / перемещение.  
Покупается: нет.  
Теги ролей: нет.  
Смысл: универсальный инструмент против рандома кубика.

Базовая активация:

```text
Игрок может заплатить фиксированную цену.
После оплаты игрок переносится на любую выбранную клетку на поле.
Клетка назначения активируется сразу.
```

Антиабуз:

```text
Нельзя использовать Такси, чтобы приехать на другое Такси и сразу снова использовать Такси.
```

## 7.11. Тюрьма

Тип: отдельная клетка / место наказания.  
Покупается: нет.  
Теги ролей: нет.  
Расположение: первый круг.

Эффект:

```text
Если игрок отправляется в Тюрьму:
- он перемещается на клетку Тюрьма на первом круге.
- теряет опыт.
- теряет активную роль.
- ход не пропускается.
```

Отличие от Больницы:

```text
Больница = откат на первый круг без обязательной потери роли.
Тюрьма = откат на первый круг + минус опыт + потеря роли.
```

## 7.12. Аукцион

Тип: интерактивная PvP-клетка.  
Покупается: нет.  
Теги ролей: Мафиози, Аферист.  
Капиталист: без специального бонуса.

Смысл: торги за свободный покупаемый объект.

Базовая активация:

```text
Игрок, наступивший на Аукцион, выбирает один свободный покупаемый объект.
Все игроки могут участвовать в ставках.
Игрок с максимальной ставкой платит свою ставку в банк и получает выбранный объект.
Если никто не участвует, аукцион не проводится.
```

Мафиози:

```text
Перед началом торгов выбирает одного игрока.
Этот игрок не может участвовать в текущем аукционе.
```

Аферист:

```text
Если Аферист выигрывает аукцион:
- получает выбранный объект.
- дополнительно получает ставку игрока, занявшего второе место.

Если второго участника нет:
- дополнительного бонуса нет.
```

Что нужно определить позже:

```text
- правила повышения ставки;
- тайбрейк;
- минимальная ставка;
- список объектов, доступных для аукциона;
- может ли выбранный объект быть с другого круга или только с текущего.
```

## 12. Обязательные системные клетки, которые еще нужно спроектировать

### 12.1. Клетка роли

Нужна, потому что роли — главная фишка игры.

Варианты:

```text
Выбери одну свободную роль.
```

или:

```text
Смени текущую роль на одну из доступных.
```

Нужно определить:

```text
- можно ли взять роль, если уже есть роль;
- можно ли отказаться от текущей роли;
- сколько длится роль;
- что происходит при потере роли;
- как часто роль-клетки должны встречаться.
```

### 12.2. Gate / переход между кругами

Нужна клетка для перехода между кольцами.

Вариант:

```text
Если хватает опыта/денег/условий, перейди на следующий круг.
```

Нужно определить:

```text
- условия перехода на 2 круг;
- условия перехода на 3 круг;
- можно ли спускаться добровольно;
- что происходит при откате из 3 круга в Больницу/Тюрьму.
```

### 12.3. Рынок

Простая покупочная клетка.

```text
Можешь купить один свободный объект на этом круге по обычной цене.
```

Отличие от Аукциона:

```text
Рынок = обычная покупка.
Аукцион = PvP-торги.
```

## 17. Core design summary for agent

Use these constraints when implementing:

```text
- Game is turn-based.
- Server/backend must be authoritative.
- Roles are temporary, unique and exclusive.
- Role effects happen only on cell activation.
- Ownership effects do not depend on owner role.
- No long passive role-based ownership bonuses.
- Do not skip turns for Hospital/Jail; use spatial setback instead.
- Hospital is first-ring setback.
- Jail is first-ring setback + experience loss + role loss.
- 2 scandals remove active role.
- Крыша blocks certain harsh effects by being consumed.
- Exact numbers are placeholders and must be data-driven.
- Put balance values in JSON/config, not hardcoded.
```
