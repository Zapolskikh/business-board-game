# Frontend: браузерный 2D-клиент

React + TypeScript + Vite. Максимально простой клиент: рисует поле из трёх кругов
(SVG), панель игроков, кнопки действий и лог. Вся логика — на backend; клиент
только отправляет действия и отображает пришедшее состояние.

## Установка и запуск

```bash
cd frontend
npm install
npm run dev
```

Откроется <http://127.0.0.1:5173>. Запросы к `/api` проксируются на backend
(<http://127.0.0.1:8000>) — сначала запустите backend (`uvicorn app.main:app --reload`).

## Что умеет прототип

- Создать игру: 2–6 игроков (человек/бот), выбор поля (board_60 / board_72), seed.
- Ходить: бросок кубика, выбор в интерактивных клетках (покупка, роли, PvP и т. д.).
- «Сыграть ход бота» — авто-проигрывание хода для игрока-бота.
- Панель игроков: деньги, опыт, роль, скандалы, Крыша, капитал.
- Лог событий с иконками.
- Встроенный симулятор баланса (вкладка на стартовом экране): прогон N партий
  ботами и таблица win-rate по ролям.

## Структура

```text
src/
  api.ts            # тонкий клиент REST
  types.ts          # типы состояния (зеркало backend)
  App.tsx           # корневой компонент, оркестрация
  components/
    Board.tsx       # SVG-поле из 3 колец
    PlayerPanel.tsx # карточки игроков
    ActionPanel.tsx # кнопки действий / решения
    EventLog.tsx    # лог событий
    GameSetup.tsx   # экран создания игры
    SimPanel.tsx    # симулятор баланса
  styles.css
```

## Как расширять UI

Новые типы клеток и роли подхватываются автоматически: клиент берёт метаданные
(цвета, названия, теги) из `/api/meta`, а решения рисуются из `pending_decision`,
который присылает backend. Обычно менять фронтенд под новую клетку не требуется —
она появится на поле и её интерактив отработает через общий механизм решений.
